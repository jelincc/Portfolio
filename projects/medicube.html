문제 정의

설명 없이는 연결할 수 없었다

신규 뷰티 디바이스 출시로 처음 도입된 기능이라 4050세대 중심 고객이 설명 없이도 바로 쓸 수 있어야 했습니다. 출시 후에는 자동연동을 오인지한 고객의 CS 문의가 이어졌고, 디바이스 등록자 중 실제 연동을 사용하는 비율은 약 45%에 그쳤습니다. 디바이스를 여러 대 가진 고객은 배너가 하나만 노출돼 기존 연결을 해제하고 다시 등록하는 일을 반복했습니다.

대상

에이지알 디바이스 보유 고객 (4050세대 중심)

상황

신규 디바이스 출시로 블루투스 연동 최초 도입

고통

자동연동 오인지로 CS 문의, 다중 디바이스 재등록 반복

성공 기준: 설명 없이 이해 가능한 한손 조작 UI, 연동 사용률 상승, CS 문의 감소
Pain Point · 개선 방향

출시 후 발견한 2가지 문제와 해결 방향

01

Pain Point:

자동연동을 오인지해 CS 문의가 이어지고, 등록자 중 45%만 실제 연동을 사용

해결 방향:

전원 켜진 디바이스를 자동 인식해 퀵연결을 바로 제안하면 사용률이 오를 것

적용 후
디바이스 등록자 중 연동 사용 약 20% 증가
02

Pain Point:

디바이스를 여러 대 등록해도 배너가 하나만 노출돼 해제와 재등록을 반복

해결 방향:

홈에 연결 가능한 디바이스 전체를 목록으로 보여주고, 연결된 디바이스는 1뎁스에서 바로 제어하게 하면 반복 등록이 줄 것

적용 후
연동 비율 약 16%p 이상 증가 · 문의 약 20% 이상 감소
근거 기반 설득

VOC · 사용성 테스트

내부 커뮤니티 문의의 34%, CS 문의의 30%가 블루투스 연동 관련이었습니다. 2뎁스 이상으로 들어가면 진입장벽이 급격히 높아지는 4050세대의 특성을 감안해, 연결된 디바이스는 최대한 얕은 뎁스에서 제어할 수 있도록 설계 방향을 잡았습니다.

핵심 인사이트: 문제의 절반 이상은 "몰라서"가 아니라 "먼저 물어보지 않아서" 생겼습니다. 매번 사용자가 연결 방법을 찾아야 했던 구조를, 디바이스가 켜지면 앱이 먼저 인식하고 제안하는 구조로 뒤집는 것이 핵심 방향이었습니다.
의사결정

자동 인식 + 1뎁스 컨트롤

자동 인식 · 퀵연결

주변에 전원이 켜진 디바이스가 있으면 자동으로 인식해 바텀시트로 연결을 제안.

홈 디바이스 목록

연결 가능한 디바이스를 홈에서 전체 목록으로 노출. 여러 대를 등록해도 배너 하나로 막히지 않음.

1뎁스 컨트롤

이미 연결된 디바이스는 목록에서 한 번의 진입으로 바로 제어 화면 도달.

UX 라이팅 분리

"확인"이 목적인 문안과 "연결"이 목적인 문안을 구분해 다음 행동을 명확히 안내.

설계 원칙: 4050세대는 2뎁스 이상 들어가는 순간 이탈 가능성이 크게 높아집니다. 기능을 더 넣기보다 이미 연결된 디바이스까지 도달하는 뎁스를 줄이는 것을 최우선으로 삼았습니다.
